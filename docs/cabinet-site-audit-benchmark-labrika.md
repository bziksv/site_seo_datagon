## 1. Позиционирование

Labrika продаёт **комплексный SEO/техаудит** как ядро продукта: краул сайта → сотни факторов → приоритеты → задачи/отчёты. Вокруг — позиции, конкуренты, ИИ-контент, реклама, мониторинг изменений.

Заявленные цифры (маркетинг, 2026):

| Заявление | Где |
|-----------|-----|
| 230+ / 270+ факторов | главная, SEO-аудит |
| SEO-аудит + ещё ~68 параметров техаудита | /seo-audit |
| Приоритеты ошибок по 4 группам значимости | /seo-audit |
| Расписание полного аудита | /seo-audit |
| Печать DOCX + таблицы XLSX, HTML-шаринг | /seo-audit |

**Вывод для Titlo:** аудитория та же (SEO, агентства, приёмка сайта, падение трафика). У нас уже есть **фрагменты** аудита разрозненными модулями — не хватает единого краула + сводного отчёта с приоритетами.

## 2. Что проверяет SEO-аудит Labrika (по публичной витрине)

Сгруппировано по смыслу (не официальная таксономия UI).

### 2.1. Индексация и доступность для ПС

- Число страниц в индексе Яндекс / Google  
- Закрытие от индексации: robots, noindex, nofollow, rel=canonical (+несколько факторов)  
- Посадочные в sitemap / потерянные посадочные / смена URL  
- Текст в noindex  

**Уже есть в Titlo (частично):** проверка индексации и сниппетов (`/index-check`), HTTP-заголовки, мета-теги.

### 2.2. Мета и заголовки

- TITLE = DESCRIPTION / TITLE = H1 / DESCRIPTION = H1 / H1 = H2…  
- Дубли TITLE / DESCRIPTION  
- Короткие TITLE / DESCRIPTION  
- Переспам в META и H1  

**Уже есть:** мониторинг мета-тегов, HTML-редактор.

### 2.3. Контент и спам-фильтры

- Плагиат / неоригинальный текст  
- Переспам в тексте, биграммы/триграммы («тошнота»), Баден-Баден-ориентир  
- Много strong, повторы слова в предложении, заспамленность заголовками  
- Тощие страницы  
- Взрослый / негативный контент  

**Уже есть:** анализ текста, Есенин, уникальность.

### 2.4. Ссылки и структура

- Внутренние ссылки на посадочные / дубли одной ссылки на странице  
- Исходящие на другие сайты  
- Страницы без исходящих внутренних ссылок  
- Похожие страницы  

**Слабо в Titlo:** нет полноценного краулера и графа перелинковки.

### 2.5. Сниппеты и выдача

- Сниппеты в Яндекс и Google  
- (в обзорах) подсветка, откуда текст попал в сниппет  

**Уже есть:** index-check сохраняет title + snippet из выдачи.

### 2.6. «Уникальные» для Лабрики акценты

- Вероятные **аффилиаты**  
- Телефон / адрес на коммерческих страницах  
- Соответствие запросов посадочным  
- Коммерческие факторы (отдельный инструмент на главной)  

### 2.7. Технический аудит (отдельный продукт, входит «ещё 68» в SEO-пакет)

Публично: факторы, мешающие ПС анализировать сайт (доступность, HTML, robots/sitemap, дубли, битые ресурсы и т.п. — типичный краулер уровня Screaming Frog / Netpeak).

## 3. UX отчёта (как продают)

1. **Сводный отчёт** — проблемы по 4 уровням приоритета.  
2. **Детальные отчёты** по отдельным параметрам.  
3. **Динамика** — график числа ошибок по разделам.  
4. **Расписание** полного аудита.  
5. **Экспорт** DOCX + XLSX, **шаринг** HTML-ссылкой клиенту/исполнителю.  
6. Связка с **задачами** (ИИ превращает аудит в план работ) — на главной.

**Принцип:** аудит = не таблица «200 красных галочек», а **очередь работ** с приоритетом и шарингом.

## 5. Gap: что Titlo уже закрывает кусками vs чего нет

| Блок аудита | Titlo сейчас | Gap |
|-------------|--------------|-----|
| Индексация URL | index-check | нет краула всего сайта + сверка с sitemap |
| Мета | meta-tags | нет автокраула всех URL проекта |
| Контент / спам | text analyzer, Есенин | нет массовой проверки всех страниц сайта |
| HTTP / ответы | http-headers | пакет до 500, но не краул по ссылкам |
| Сниппеты | index-check | нет истории сниппетов по сайту |
| Перелинковка / дубли URL / canonical | — | **нет** |
| Аффилиаты / взрослый контент | — | **нет** |
| Единый отчёт + приоритеты + шаринг | — | **нет** (главный gap) |
| Расписание аудита | — | **нет** |

**MVP-гипотеза для Titlo «Аудит сайта»:**

1. Проект = домен + список URL (sitemap / ручной / импорт).  
2. Краул или пакетная проверка существующих модулей (meta, headers, index, text) в одном job.  
3. Сводка: критично / важно / средне / низко.  
4. Шаринг отчёта + CSV/XLSX.  
5. Позже: расписание, граф ссылок, аффилиаты.

Не пытаться сразу «270 факторов» — закрыть **склейку** того, что уже есть, плюс краул и приоритеты.
