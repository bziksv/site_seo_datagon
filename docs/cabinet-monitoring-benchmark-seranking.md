**Маркетинг:** https://seranking.com/ (кнопка «Request a demo» — лид-форма, не продукт)

**Полная карта всех страниц продукта:** [seranking-product-map.md](./seranking-product-map.md) (аккаунт, research-инструменты, меню проекта, Local/SMM, индекс URL).  
**Второй бенчмарк (Topvisor):** [topvisor-product-map.md](./topvisor-product-map.md) · [cabinet-monitoring-benchmark-topvisor.md](./cabinet-monitoring-benchmark-topvisor.md).

Конспект **этого файла** сфокусирован на **Rank Tracker / список проектов / позиции** — ближайший аналог нашему `/monitoring` и `/monitoring-v2`.

## 1. Архитектура продукта (логика экранов)

SE Ranking строит UX вокруг **трёх уровней глубины**:

| Уровень | URL (паттерн) | Назначение |
|---------|---------------|------------|
| **Аккаунт** | `admin.dashboard.html` | Все проекты + агрегированный график + группы |
| **Проект (хаб)** | `admin.site.overview.site_id-{id}.html` | Виджеты: позиции, аудит, GSC, конкуренты, AI Search |
| **Модуль «Позиции»** | `admin.site.rankings.site_id-{id}.html#/{view}` | Обзор / Подробный вид / Исторические данные |

**Принцип:** на списке проектов — только **дайджест**; полные таблицы ключей и регионов — **отдельные экраны**, не вложенные в строку списка на полную высоту.

Hash-роутинг внутри позиций:

- `#/overview` — дашборд метрик и распределение TOP
- `#/detailed` — таблица ключей с колонками по датам
- `#/history` — историческая матрица (длинный горизонт)

## 2. Глобальная навигация

### 2.1. Верхняя полоса (модули аккаунта)

- Позиции (Rank Tracker)
- Аудит сайта (Проекты)
- Анализ конкурентов
- Анализ ключевых слов
- Отчёты, SMM-модуль, биллинг, White Label

### 2.2. Боковая панель (иконки)

- **Проекты** (активно на дашборде)
- Инструменты анализа
- Бэклинки
- Аудит
- **AI-поиск** (badge New)
- Контент-маркетинг, локальный маркетинг
- Отчёты, пакет для агентств, API

### 2.3. Переключатель проекта

В контексте проекта — дерево: домен → группы (Магазины, СМИ, Медицина) → подпроекты.  
Боковое меню проекта дублирует разделы: Обзор проекта, Позиции (3 подпункта), Аналитика, Конкуренты, AI-трекер, Аудит, Бэклинки и т.д.

**Вывод для Datagon:** не смешивать «список всех проектов» и «глубокий модуль одного проекта» в одном scroll — отдельный route или явный drill-down.

## 3. Дашборд проектов (`admin.dashboard.html`)

### 3.1. Верхний блок — мультипроектный график

**Вкладки метрик** (переключают Y и легенду):

- Средняя позиция (по умолчанию)
- Прогноз трафика
- Видимость
- % ключей в ТОП-10
- Присутствие упоминания / ссылки (AI)

**Фильтры времени:** Неделя | Месяц | 3 месяца | 6 месяцев | Все  
**Группировка:** «Сгруппировать по: Дням» (dropdown)

**Легенда:** цвет = проект или **группа проектов** (Agrus, СМИ, Магазины, Медицина, Объявления).

**Поведение:** график грузится асинхронно (спиннер); ось Y для позиции **инвертирована** (меньше число = лучше).

### 3.2. Toolbar списка

| Элемент | Функция |
|---------|---------|
| Поиск | Фильтр по имени проекта |
| Экспорт | Выгрузка таблицы |
| Создать проект | Онбординг нового сайта |
| Обновить данные | Пересъём позиций (с dropdown) |
| Колонки | Показ/скрытие столбцов |
| Копировать | Bulk / clipboard actions |

Счётчик выбора: «Выбрано N проектов (M ключевых слов)».

### 3.3. Таблица «Проекты»

**Режим по умолчанию — таблица**, не плитки (переключатель вида есть в заголовке секции).

Типичные колонки (настраиваемые через «Колонки»):

| Колонка | Пример | Примечание |
|---------|--------|------------|
| Чекбокс | — | Массовые действия |
| Проект | favicon + домен | Клик по имени — на сайт клиента; отдельно иконка графика → позиции |
| ТОП 5 / 10 / 30 | `0 / 0 / 9` | Три числа в одной ячейке |
| Ключевые слова | `28` | Ссылка в модуль |
| Промпты (AI) | Add | Отдельная фича |
| Ср. позиция | `64` + **−4** | Дельта цветом (красный вниз, зелёный вверх) |
| В индексе | Google `10.6K` | Индексация |
| Дата проверки | `28 Май, 2026` | |

**Раскрытие строки (expand):** не полная таблица ключей, а **дочерние регионы/локали** той же структуры колонок (например «Киев, Украина» с TOP и видимостью `77/100`). Это аналог наших child-rows, но **компактный**.

**Меню строки (⋮):**

- Обзор проекта
- Позиции
- Бэклинки
- План продвижения
- Аудит сайта

Быстрый переход в модуль без двойного клика по домену.

### 3.4. Блок «Группы проектов»

Отдельная таблица под списком проектов:

- Группы: Магазины, СМИ, Медицина, Объявления
- Те же метрики TOP / видимость / дата
- «Развернуть группы», «+ Новая группа»
- Вкладки вверху графика: **Проекты | Группы** — переключение агрегации на графике

**Вывод:** агентствам важны **папки/группы**; у Datagon пока только плоский список + теги в меню.

## 4. Модуль «Позиции» проекта (`admin.site.rankings…`)

Общий **chrome** для всех трёх видов:

- Breadcrumb: `{домен} > Позиции > {Обзор|Подробный вид|Исторические}`
- Фильтр ПС: «Все ПС» или конкретная
- Диапазон дат (date_range)
- Data Studio, Экспорт, Настройки
- «Добавить ключевые слова», «Обновить данные»
- Прогресс: «100% обработано» (статус съёма)

Подменю: **Обзор | Подробный вид | Исторические данные**

---

### 4.1. Обзор (`#/overview`)

**KPI-карточки (3 в ряд):**

1. **Видимость** — % + sparkline  
2. **Средняя позиция** — число + дельта (▲6) + line chart за период  
3. **Органический трафик** — CTA «Подключить Google Analytics»

**Блок «Распределение топ-позиций»:**

- Мини-KPI: TOP 1, TOP 2–3, TOP 4–5, TOP 6–10 (число + микрографик)
- Большой **stacked area chart** по времени
- Легенда: TOP 1, 2–3, 4–5, 6–10, 11–30, … TOP 100
- Периоды: Текущее | 7Д | 1М | 3М | 6М | 1Г | 2Г

**Ниже — виджеты-дайджесты** (не одна таблица):

| Виджет | Содержание |
|--------|------------|
| Улучшились позиции | Топ ключей с ростом + «Посмотреть все (28)» |
| Ухудшились | Топ падений + счётчик |
| Потеряли TOP | Ключи, выпавшие из TOP |
| Новые в TOP | Вошли в TOP |
| Целевые URL | Сниппеты страниц в выдаче |
| Конкуренты в TOP | Домены конкурентов (57) |
| Заметки | 45 заметок по проекту |

Фильтр: «Все группы» ключей, вид «Список».

**Вывод:** «Обзор» = **storytelling + графики**; операционная работа — в «Подробном виде».

---

### 4.2. Подробный вид (`#/detailed`)

**Полоса KPI над таблицей:**

- % в TOP-3, TOP-10 (со sparkline)
- Средняя позиция: `28`
- Счётчики динамики: улучшилось / ухудшилось / без изменений / новые / выпали (числа с дельтами)

**Контекст региона:** «Google Ukraine, Киев…» + переключатель локали RU.

**Toolbar таблицы:**

- Поиск по ключам
- Вид: «Список» (dropdown — возможны другие представления)
- Копировать, **Фильтры**, **Колонки**
- Пагинация: 25 / 50 / **100** на страницу

**Таблица ключевых слов:**

| Колонка | Поведение |
|---------|-----------|
| Ключевое слово | Текст + ссылка |
| URL | Иконка целевой страницы |
| Частотность | 480, 1.6K |
| Даты (26 мая, 27 мая, …) | Позиция + **дельта** (+73 зелёный, −1 красный) |
| SERP features | Иконки (в расширенных колонках) |

Клик по **цифре позиции** → кэш SERP за день проверки (онбординг это подчёркивает).

**Вывод для Datagon:** это ближе всего к классическому `/monitoring/{id}`; список проектов такую таблицу **не дублирует**.

---

### 4.3. Исторические данные (`#/history`)

- Та же toolbar (поиск, группы ключей, фильтры, колонки, 100/стр)
- **Широкая матрица:** ключ × много дат (горизонтальный скролл)
- Без отдельного date_range в toolbar (фокус на полной истории)
- Регион: один выбранный (Киев)

**Вывод:** история — **отдельный режим**, не смешивается с «оперативным» detailed за 2–3 дня.

## 6. Визуальный язык и UX-паттерны

### 6.1. Цвета и семантика

- **Зелёный** — улучшение позиции / рост TOP
- **Красный** — ухудшение
- Нейтральный серый фон, белые карточки, тёмный sidebar (#2c3e50-подобный)
- Primary blue для CTA и активных вкладок
- Графики: палитра по проектам/группам (не по случайным KPI)

### 6.2. Типографика и плотность

- KPI — **крупные числа** (64, 0%, 28)
- Дельты рядом с числом, не в отдельной колонке без контекста
- Таблицы — умеренная плотность, zebra rows, highlight последнего дня

### 6.3. Производительность (наблюдаемое)

- График на дашборде — lazy load (спиннер)
- Списки — пагинация 100 строк
- Нет «загружено 86/86» — вместо этого **% обработано** съёма или статус в toolbar

### 6.4. Онбординг

Пошаговые тултипы (6 шагов на дашборде, 5 в позициях): графики → таблица → меню строки.  
Можно закрыть (×).

## 7. Сравнение с Datagon monitoring v2 (2.8-dev)

| Аспект | SE Ranking | Datagon v2 сейчас | Рекомендация |
|--------|------------|-----------------|--------------|
| **Главный вид списка** | Таблица | Плитки + опция таблицы | **Таблица по умолчанию** для 50+ проектов |
| **График на списке** | Да, переключаемые метрики | Chart.js дашборд (3 графика) | Оставить, сделать **сворачиваемым**; переключатель метрик как у SER |
| **Child-rows** | Регионы, компактно | Полная таблица регионов/дней | В плитках — **только TOP+Δ**; полная таблица — в таблице/`+` или на странице проекта |
| **Глубина** | Отдельные URL | Раскрытие на месте | Клик по проекту → `/monitoring/{id}` или v2 drill-down |
| **Группы** | Папки + график по группам | Нет | Backlog: теги/папки проектов |
| **Дельты** | Везде, цвет | Есть в таблице | Усилить в таблице и на карточках-digest |
| **Видимость** | Отдельная метрика | TOP10 % | Добавить «видимость» в снимок, если есть формула |
| **Индексация** | Колонка «В индексе» | Нет в v2 | Опциональная колонка из снимка/API |
| **Обзор vs операции** | Разделены | Смешаны в одной странице | Разделить **дайджест** (карточка) и **операции** (таблица) |
| **Исторические данные** | Отдельный hash-view | Child-rows по месяцам | Не тянуть всю историю в список |
| **Экспорт / колонки** | Да | Частично (таблица) | Паритет с classic monitoring toolbar |
| **Заметки по проекту** | 45 заметок, виджет | Нет | Низкий приоритет |

## 8. Что взять в roadmap monitoring v2 (приоритет)

### P0 — быстрый выигрыш UX

1. **Таблица — режим по умолчанию** (localStorage можно оставить, default = table).
2. **Сворачиваемый дашборд** (chevron), чтобы таблица была above the fold.
3. **Карточки — digest-only:** домен, TOP5/10/30, ср.поз.+Δ, слова, дата; без полного child-table.
4. **Цветовые дельты** единообразно (как SER): зелёный/красный, не btn-info.

### P1 — паритет с SER list

5. Колонки **ТОП 5/10/30** одной ячейкой `a / b / c`.
6. **Expand row** = только регионы (как SER), не keyword grid.
7. Меню строки: Мониторинг, настройки, отчёт (если есть).
8. Переключатель метрик на дашборде: средняя позиция | TOP10% | видимость (из снимка).

### P2 — структура продукта

9. Отдельная страница **«обзор проекта»** (виджеты из снимка + ссылки).
10. **Группы проектов** (папки) для агентств.
11. Исторический вид — отдельная вкладка на странице проекта, не в списке.

### Не копировать слепо

- AI Search / промпты / Data Studio — вне scope мониторинга Datagon.
- Лид-форма «Request a demo» — не путать с product demo (`login.demo.html`).
